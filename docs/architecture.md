## 二、各层职责

### 1. 数据采集层

负责采集用户行为数据，所有数据均在浏览器本地处理，不上传网络。

| 采集源 | 事件类型 | 处理方式 | 用途 |
|--------|---------|---------|------|
| 键盘 | keydown | 节流 500ms | 分析工作强度 |
| 鼠标 | mousemove | 节流 500ms | 分析活跃度 |
| 页面 | visibilitychange | 即时 | 识别离开状态 |
| 计时器 | 状态变化 | 即时 | 记录工作时段 |

### 2. Service 层

#### 2.1 StorageService
- 统一封装 LocalStorage 操作
- 添加 'workease_' 前缀
- 处理 QuotaExceededError
- 数据持久化管理

#### 2.2 SmartRhythmService（底层分析引擎）
- 行为采样与分析
- 三状态状态机（Focus/Normal/Relax）
- 滞后机制防止状态频繁切换
- 智能提醒策略计算

#### 2.3 WorkSenseService（工作状态感知）
- 包装 SmartRhythmService
- 提供增强的 AI 分析能力
- 健康收益预测 (`estimateBenefit()`)
- AI 洞察生成 (`generateInsight()`)
- 动态解释文本生成

#### 2.4 HealthService（健康评分）
- 多维度健康评估
- 实时评分计算
- 趋势追踪

#### 2.5 SmartCoachService（智能教练）
- 个性化动作推荐
- 推荐理由生成
- 训练记录管理

#### 2.6 HealthRecordService（健康记录）
- 每日记录生成
- 时间轴事件管理
- 成就徽章系统
- AI 分析报告

### 3. UI 层

#### 3.1 WorkSenseCard
- 实时显示当前工作状态
- 连续工作时间（数字滚动动画）
- AI 分析文本展示
- 今日专注统计
- 健康收益预测

#### 3.2 HealthScoreCard
- 健康评分展示（数字滚动动画）
- 五维度评分详情
- 改善建议

#### 3.3 SmartCoachCard
- 推荐动作卡片
- AI 解释文本
- 开始训练/稍后提醒/换一个按钮

#### 3.4 AI健康记录页面
- 今日健康概览
- 时间轴记录
- AI 分析
- 今日亮点/需要改善
- 明日目标
- 成就徽章
- 分享卡片

## 三、状态机设计

WorkSense™ 使用三状态状态机来感知用户工作状态：

```
                    ┌─────────────────────────────┐
                    │         状态转换规则         │
                    └─────────────────────────────┘

        Focus (深度专注)                            
            ↑                                     
            │ 连续高频操作(≥5分钟) + 页面可见度高       
            │                                     
    ┌───────┴───────┐                             
    ↓               ↓                             
  Normal          Relax                            
(正常办公)       (休息中)                           
    ↓               ↑                             
    │ 操作频率下降   │ 长时间无操作(≥10分钟)         
    └───────────────┘                             

滞后机制：状态转换需持续 ≥5 分钟才确认，避免频繁跳变
```

### 状态判断规则

| 状态 | 键盘频率(次/分钟) | 鼠标频率(次/分钟) | 页面可见度 | 连续时长 |
|------|------------------|------------------|-----------|---------|
| Focus | ≥30 | ≥60 | ≥80% | ≥5分钟 |
| Normal | 5-30 | 10-60 | ≥50% | - |
| Relax | ≤5 | ≤10 | <50% 或 ≥10分钟无操作 | ≥5分钟 |

## 四、智能提醒策略

基于当前工作状态动态调整提醒时机：

| 当前状态 | 基础提醒时间 | 调整策略 | 最大延迟 |
|---------|-------------|---------|---------|
| Focus | 45分钟 | +10~20分钟 | 65分钟 |
| Normal | 45分钟 | 不变 | 45分钟 |
| Relax | 立即 | 建议拉伸 | - |

### 智能调整规则
- 连续关闭提醒 ≥3 次 → 下次提醒延迟 +15分钟
- 连续完成拉伸 ≥3 次 → 奖励：允许更长专注时间
- 最近5分钟操作频率下降50% → 进入准备提醒状态

## 六、存储设计

所有数据通过 StorageService 存储在 LocalStorage 中，使用 'workease_' 前缀：

| 存储键 | 内容 | 过期策略 |
|--------|------|---------|
| workease_settings | 用户设置 | 永久 |
| workease_health_data | 健康数据 | 永久 |
| workease_rhythm_data | 节奏分析数据 | 保留最近30天 |
| workease_worksense_insights | AI洞察历史 | 保留最近10条 |
| workease_health_records | 健康记录 | 保留最近30天 |
| workease_badges | 成就徽章 | 永久 |
| workease_today_events_* | 今日事件 | 每日 |

## 七、技术特点

### 1. 零依赖架构
- 纯原生 JavaScript，无需外部框架
- 静态 HTML，无需构建工具
- 所有依赖通过 CDN 引入

### 2. 隐私保护
- 所有分析均在浏览器本地完成
- 无云端数据上传
- 不记录键盘输入内容
- 不上传鼠标轨迹

### 3. 模块化设计
- Service 层与 UI 层分离
- 业务逻辑集中在 Service 文件
- 组件独立封装，可复用

### 4. 响应式设计
- 手机/平板/电脑/超宽屏适配
- 使用 Tailwind CSS 响应式断点

### 5. 性能优化
- 事件节流处理
- 骨架屏预加载
- 动画使用 requestAnimationFrame
- 组件按需渲染

## 八、扩展性设计

### 未来扩展点
1. **AI 聊天助手**: 在 WorkSenseService 基础上扩展对话能力
2. **语音提醒**: 集成 Web Speech API
3. **个性化训练计划**: 基于历史数据分析
4. **多日学习**: 增强 AI 模型的学习能力
5. **硬件集成**: 支持智能手环/手表数据

### 扩展方式
- 新增 Service 文件，不修改已有业务逻辑
- 新增 UI 组件，通过事件总线与现有模块通信
- 保持数据模型向后兼容
